Blockchain Workshop Notes

Introduction to Blockchain Concepts

  • Blockchain: A decentralized, distributed ledger technology for storing transaction records in immutable blocks.
  • Decentralization: Eliminating a central authority to improve security and transparency.
  • Distributed Ledger: A shared database replicated across nodes in a network.
  • Hashing: A cryptographic method to map data to fixed-size values securely.

SHA-256

Key characteristics of the SHA-256 hashing algorithm:

  1. Deterministic: Same input produces the same output every time.
  2. Collision Resistance: Difficult to find two inputs that produce the same hash.
  3. Fast: Quick computation ensures efficiency.
  4. One-Way: Impossible to reverse-engineer input from the hash.

A valid hash meets specific criteria, such as having a fixed number of leading zeros.

  • Hash value = Transaction data + Nonce (magic number).
  • Each block in the blockchain includes:
    • Previous block’s hash
    • Current transaction data
    • Nonce
    • Current block’s hash

Nodes and Miners

  • Nodes: Validate transactions and maintain the blockchain.
  • Miners: Add blocks to the blockchain by solving complex mathematical puzzles.

Cryptology in Blockchain

  • Symmetric Encryption: Same key for encryption and decryption.
  • Asymmetric Encryption: Uses a private key to sign data and a public key to verify signatures (used in blockchain).

Validation Techniques by Nodes

Nodes validate data through digital signatures:

  1. Sign: Private key + Transaction data.
  2. Verify: Ensures the validity of the signature.

Merkle Tree

A Merkle tree enhances blockchain security by hashing transaction data into a tree structure rather than a linear chain.

Consensus Algorithms

Proof of Stake (PoS) vs Proof of Work (PoW):

  • PoW (used in Bitcoin): Miners solve puzzles to add blocks.
  • PoS: Validators are selected based on their stake.

Key Concepts in Bitcoin and Ethereum

Bitcoin Improvement Proposal (BIP)

  • Simplifies wallet management.
  • Derives multiple private keys from a single seed.
  • Mnemonic seed phrases (e.g., “cat dog bird”) are converted into seeds using libraries.

Derivation Path:

  • m /44/60’/1’/private key
    • m and 44: Standards.
    • 60': Blockchain type (Ethereum = 60).
    • 1': Account number.

Session 2: Advanced Blockchain Topics

RPC Nodes

  • Broadcast transactions.
  • Prioritize miners with high fees in mempool.
  • Example blockchain performance:
    • Bitcoin TPS: ~7
    • Ethereum TPS: ~12-15

What is an RPC Node?

  • A computer running blockchain client software to process remote requests.
  • Types of Ethereum nodes:
    • Light nodes, full nodes, archival nodes.

Further reading: What is an RPC node?

Zero-Knowledge Rollups (zk-Rollups)

Layer-2 scaling solution:

  • Computation occurs off-chain.
  • Transaction data is stored on-chain.
  • Validated via zero-knowledge proofs.

Read more: Zero-Knowledge Proof.

Turing Complete Systems

A system capable of solving any computational problem (e.g., Ethereum).

Solidity Contracts

  • Written in Solidity language.
  • Automates self-executing contracts.

Features:

  1. Immutable (unchangeable).
  2. Transparent (publicly visible).
  3. Self-executing (trigger-based).

Use Cases:

  • Supply chain, voting systems, identity verification.

Tools: Remix IDE for Solidity.

Decentralized Applications (dApps)

  • Definition: Applications without centralized control.
  • Example dApp: Counter DApp on GitHub.

RPC Protocol in Blockchain

  • Uses JSON-RPC for client-server communication.

Ethereum JSON-RPC Methods:

  1. Gossip Methods: Track blockchain head.
  2. State Methods: Retrieve current blockchain state.
  3. History Methods: Access historical data.
