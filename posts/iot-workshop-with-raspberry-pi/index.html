January 4, 2025 - IoT Workshop with Raspberry Pi

Today was one of the most educational days I’ve had in a while. The IoT Workshop for Raspberry Pi, divided into a morning and an evening session, was packed with insights about IoT technologies, hardware, and practical applications. I took extensive notes and captured every key point and demonstration. Here’s the complete breakdown of the day:

Morning Session: Foundations and Hardware of IoT

Time: 9:00 AM to 12:00 PM

Topics Covered

  1. Prominent IoT Platforms:

    • Azure IoT
    • IBM IoT
    • AWS IoT
      Each of these platforms offers robust services to handle data processing, networking, and IoT device management.

  2. IoT Hardware and Concepts:

    • LoRa and Zigbee: Widely used protocols supporting a large number of connections over long distances with minimal bandwidth.
    • IoT Applications: Demonstrated how IoT can monitor and measure various environmental variables such as temperature, humidity, light, and pressure.

Four Core Components of IoT

  1. Networking: The backbone of IoT for connecting devices.
  2. Sensors: Devices that collect data from the environment.
  3. Actuators: IoT devices that perform actions, like switching on lights or moving motors.
  4. Data Analytics: Focused on processing and interpreting IoT data:
    • Descriptive Analytics: Provides insights into past events.
    • Predictive Analytics: Helps forecast future trends.
    • Real-time Analytics: Facilitates decision-making on the fly.

Hardware Notes for IoT Beginners

  • Single-Board Computers (SBC): Raspberry Pi stands out due to its multitasking abilities and microprocessor integration:
    • SBC > Microcontrollers in tasks requiring multi-threading (like running AI models or handling complex logic).
  • Popular Boards for IoT:
    • Intel Galileo: Offers Intel architecture compatibility.
    • Nvidia Boards: Equipped with GPUs and CUDA support for running AI models efficiently.
    • Azure Tinkerboard: Provides advanced capabilities and IoT edge integrations.

Recommended Raspberry Pi Boards

For beginners:

  • Pi Zero
  • Pi W
  • Pi Zero W

Key Accessories for Raspberry Pi:

  • SD cards (at least Class 10)—Raspberry Pi boards do not come with built-in storage.
  • HDMI adapters, micro USB OTG cables, and USB hubs.
  • Analog sensors require ADC converters since the Pi accepts only digital signals.

Best Websites for IoT Hardware

  • RoboCraze
  • Silverline Electronics
  • Thingbits.in—Ships fast from Mumbai.

Hands-On Hardware Setup

  1. After booting the Pi with the Raspberry Pi OS Lite 32-bit, the essential steps for configuring were:
    • Enable SSH during setup.
    • Use a hotspot or network supporting 2.4 GHz only for WiFi. Avoid spaces in the hotspot name!
    • Set keyboard layout to US.
  2. By default, the login credentials are:
    • Username: pi
    • Password: raspberry

Tips and Learnings:

  • Raspberry Pi can only support up to 32 GB storage natively.
  • If Raspberry Pi shuts down, the green LED stops blinking.
  • Raspberry Pi boards run on ARM-based processors.

Evening Session: Practical Projects and Programming

Time: 2:00 PM to 5:00 PM

Node-RED Overview

Node-RED is an open-source flow-based programming tool designed to simplify IoT workflows. Installed locally via the Raspberry Pi setup at localhost:1880, it allows for easy integration and control of connected devices.

Modules Installed:

  • Node-RED Dashboard: For creating user-friendly interfaces with buttons, switches, and gauges.
  • Debug tools for real-time monitoring of data types (booleans, strings, integers, etc.).

Node-RED Example Setup

We designed a smart home layout:

--TAB 1  
   |_GROUP 1  
   |    |_ON/OFF Buttons  
   |    |_Switches for Hall and Garden Lights  
   |_GROUP 2  
   |    |_Gauges: Temperature & Humidity

Protocols in IoT

MQTT (Message Queue Telemetry Transport)

A lightweight protocol for transmitting data over TCP/IP.

Mosquitto Commands:

To publish messages:

mosquitto_pub -t topic_name -m "Hello World"

To subscribe and receive messages:

mosquitto_sub -t topic_name

REST API

A flexible, web-based protocol similar to HTTP, used widely in IoT applications.

Application Session Projects

  1. Integrating Sensors with GPIO Pins:

    • Developed a Blink LED project (pin 29).
    • Configured a Passive Infrared (PIR) sensor for motion detection.
    • Experimented with DHT sensors for temperature and humidity readings.

  2. Telegram Bots:
    Learned to run TeleBots using Raspberry Pi, opening up exciting IoT possibilities like sending real-time sensor data over Telegram.

  3. Advanced Dashboard Building:
    Used prebuilt templates to streamline the process. Explored hierarchical tab and group structures for smart dashboards.

  4. Node-RED Extensions:
    Imported additional nodes like MQTT broker nodes to experiment with payload communication across devices.

Key Takeaways

  1. Learned the importance of creating backups and using high-speed SD cards.
  2. Installing libraries and dependencies might vary across devices; sometimes, manual compilation is required for compatibility with ARM processors.
  3. Discovered fascinating tools like:
    • PuTTY: A Windows tool for remote terminal access.
    • HiveMQ: A free, serverless MQTT solution.

Personal Notes

The IoT session demonstrated not just how IoT devices work but also the growing potential in automation, machine learning, and real-world applications.

The Node-RED flow visualizer was an eye-opener, simplifying processes I thought would need intricate coding. I’m especially looking forward to trying more projects with Node-RED and designing intuitive dashboards for my own experiments.

It was a day packed with learning, and the enthusiasm carried forward to the Robowars event (more on that in a separate post). I can’t wait to build my own IoT solutions soon!
